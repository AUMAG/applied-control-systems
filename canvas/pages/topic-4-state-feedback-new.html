  Concepts

Please go through the following resources carefully to gain an understanding of the key concepts. We also encourage you to engage in the Online Discussion for this topic. You will begin to apply these concepts in your Workshop and Practical (details below).

Reachability

This concept introduces the important properties of reachability and controllability for a controlled dynamical system — they allow us to assess via quantitative metric whether a system has the necessary inputs to be able to appropriately control it. For instance, in a car without a steering wheel you can control the velocity, but not the left-right position on the road; or for a helicopter with a broken tail rotor, you can control lift and speed, but not yaw (spinning) motion.

Stabilisation and state feedback

This concept covers the use of feedback in a state-space system to stabilise and/or affect the properties of a dynamical system. This involves measuring the states of the system, applying a matrix of gains to those states, and passing the resulting control signals back into the system. The closed loop performance can be designed by choosing the resulting eigenvalues of the system.

Design consideration

This concept examines the relationship between the location of the eigenvalues for a system and its performance metrics such as overshoot and settling time. These are examined specifically for second-order systems in terms of analytical equations, which can often be used empirically for approximate system identification. Higher order systems are also discussed.

Integral action

Stabilising a system is not always enough — often want a system to follow a given reference exactly even if there are disturbances or modelling errors. In this concept, this is achieved through the addition of an Integral Action dynamic term, which is analogous to the "I" term in a PID controller.

Output feedback basics

This is the final concept for this topic. This concept is slightly unusual as we wanted to provide a brief overview of the next chapter in the book on "Output Feedback", which covers the common situation when you are not able to directly measure all state variables of a system. Instead we build linear "observers" with the same methodology as we build a linear controller, whose input are the system outputs (i.e., the measurements), and whose output estimates the unmeasured state variables. A specific and general form of this approach is known as the Kalman Filter, which is a topic for a more advanced course.
